# 项目运行说明

## 为什么不能直接打开index.html？

由于项目使用了ES模块（ESM），浏览器的安全策略禁止通过 `file://` 协议加载模块化的JavaScript代码。这是现代Web应用的标准安全机制，无法绕过。

## 给他人运行项目的方案

### 方案1：部署到静态托管平台（推荐，完全免费）

将 `dist` 文件夹部署到以下任一平台：

- **GitHub Pages**: 免费，适合开源项目
- **Netlify**: 免费且简单，支持拖拽部署
- **Vercel**: 免费，性能优秀
- **Cloudflare Pages**: 免费，全球CDN加速

部署后任何人都可以通过URL访问，无需安装任何东西。

### 方案2：使用启动脚本

双击运行 `启动项目.bat` 文件即可启动本地服务器，在浏览器访问 `http://localhost:5000`。

注意：这需要对方已安装 Node.js。

### 方案3：打包为Electron应用

如果需要桌面应用体验，可以将项目打包为Electron应用，可以脱离浏览器独立运行。

需要修改项目结构并安装Electron依赖。

### 方案4：使用便携式HTTP服务器

如果对方没有Node.js，可以使用以下替代方案：

1. **Python内置服务器**（如果对方有Python）：
   ```bash
   python -m http.server 8000 --directory dist
   ```

2. **简单的Go服务器**（单文件可执行程序）：
   下载 `serve.exe` 或类似工具，放在项目根目录使用。

## 推荐做法

如果是临时演示：
- 使用 `npx serve dist -l 5000` 或双击 `启动项目.bat`

如果是长期分享：
- 部署到 Netlify/Vercel 等免费托管平台，获得永久URL
